The Emojicode C Style Guide

Hint

Don’t read this guide until you intend to work on Emojicode’s source or to build a package in the programming language C.

Whitespace

  • Use soft tabs set to 4 spaces.
  • Place 1 space before the leading brace.
  • Place 1 space before the opening parenthesis in control statements (if, while etc.). Place no space before the argument list in function calls and declarations.
  • Set off operators with spaces.

Types

  • Place 1 space before and no space after the asterisk of a pointer type if it is followed by a variable name or if it is in a cast.

    Otherwise place no space before and 1 space after in an

Example:

// bad
char* string;
char * anotherString = (char*)bytes;

Thread * magicFunction();
void *doThat(int * takeMe);

// good
char *string;
char *anotherString = (char *)bytes;

Thread* magicFunction();
void* doThat(int *takeMe);

Braces

The leading brace goes always on the same line as the statement to which the block belongs.

The closing brace has always a line for its own.

Something runFunctionBlock() {
    while (thread->tokenStream < end) {
        if (thread->returned) {
            doIt(3 * 5 - 3);
        }
        else {
            crash();
        }
    }
    return NOTHINGNESS;
}

Naming Conventions

  • Use PascalCase when naming types.
  • Use camelCase when naming variables and functions.
  • Use descriptive names for variables, functions and type.

Example:

// bad
void itoa();
void integer_to_string();
void IntegerToString();

// good
void integerToString();

// bad
typedef ei uint64_t;
typedef emojicodeInteger uint64_t;

// good
typedef EmojicodeInteger uint64_t;
